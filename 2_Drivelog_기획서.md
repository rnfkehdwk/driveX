## 1. 프로젝트 비전

### 1.1 한 줄 요약
**한국 대리운전 업체 사장님과 기사님이 디지털에 익숙하지 않아도 쉽게 쓸 수 있는, 콜 배차부터 정산까지 한 번에 처리하는 SaaS 플랫폼.**

### 1.2 해결하는 문제
한국 지방의 대리운전 업체들은:
- 콜 배차를 카카오톡 단톡방으로 처리하다 누락
- 정산은 사장님이 손으로 엑셀에 적어서 매월 말 끝없이 시간 소모
- 마일리지/충성고객 관리는 머릿속에만 있음
- 기사 출퇴근/근무시간 관리가 어려움
- 기존 솔루션들은 너무 복잡하거나 디지털 약자에게 진입장벽이 높음

### 1.3 핵심 가치
1. **단순함**: 사장님이 디지털 약자여도 쓸 수 있도록 — 고정 URL + 업체코드/ID/PW만 입력하면 끝
2. **자동화**: 콜 → 운행 → 정산 → 사용료 청구까지 한 흐름으로 자동화
3. **현실 반영**: 한국 지방 대리업체 관행 (시급제, 마일리지, 결제그룹 분리, 1대2 운행 등) 그대로 지원
4. **개인정보 친화**: 현금/계좌이체 같은 비공식 결제도 정식 결제구분으로 지원

## 3. 멀티테넌트 아키텍처

### 3.1 데이터 격리
모든 핵심 데이터에 `company_id` 컬럼이 있어, **하나의 DB에 여러 업체 데이터가 공존하지만 완전히 격리**됩니다.

```
DriveLog DB
├── company_id=1 (테스트 업체)
├── company_id=3 (양양대리, 1,509,000원 마일리지 운영 중)
└── company_id=N (확장 가능)
```

### 3.2 고정 URL + 업체코드 가입
- 사장님은 항상 **같은 URL**로 접속 (`https://rnfkehdwk.synology.me:38443/admin/`)
- 로그인 시 **업체코드** + ID + PW 입력
- 기사는 사장님이 알려준 업체코드로 모바일 PWA 가입 (예: "1012")

→ 디지털 약자 친화: URL 기억 부담 ↓, 가입 절차 단순화

### 3.3 확장 모델
- **셀프 가입** 지원 (`registration_enabled` 시스템 설정)
- **무료 체험** 14일 (자동 승인 옵션 포함)
- MASTER가 가입 신청 검토 후 승인

## 4. 핵심 기능 도메인

### 4.1 콜 시스템 (calls)
**배차 관리** — 사장님이 콜을 등록하고, 기사가 수락하거나 사장님이 직접 지명.

#### 콜 생명주기
```
WAITING (대기) → ASSIGNED (배정) → IN_PROGRESS (진행 중) → COMPLETED (완료)
                                                          ↘ CANCELLED (취소)
```

#### 핵심 기능
- 출발지/도착지 자동완성 (자주 가는 주소 ⭐ 즐겨찾기 — 90일 내 top 20)
- **수동 지명**: 사장님이 특정 기사에게 콜 직접 배정 (WAITING 단계 건너뜀)
- 결제구분 미리 지정 (현금/카드/기사계좌/회사계좌/나라시/미수/마일리지)
- 도착지 NULL 가능 (운행 중 결정)

### 4.2 운행 일지 (rides)
**대리운전의 핵심 데이터** — 콜에서 시작되거나 직접 등록.

#### 입력 필드
- 출발지/도착지 (주소 + 상세주소 + 좌표)
- 운임 (총 운임, 현금 금액, 마일리지 사용)
- 결제구분 (payment_type_id)
- 기사 / 픽업 기사 (1대2 운행 지원)
- 고객 / 제휴업체 (선택)
- 메모 (기사 / 관리자)

#### 자동 처리
- 운행 종료 시 마일리지 적립 (운임의 N%, 마일리지 사용분 제외)
- 운행 종료 시 마일리지 사용 차감 (잔액 검증, 5000원 단위 검증)
- GPS 비교 검증 자동 생성

### 4.3 정산 시스템 (settlements)

#### 4.3.1 정산방식 3종 (`company_pay_settings.pay_type`)
| 방식 | 설명 | 계산 |
|---|---|---|
| **HOURLY** (시급제) | 양양대리 | rider_share = 시급 × 근무시간 |
| **PER_RIDE** (건별제) | | rider_share = 건당단가 × 운행건수 |
| **COMMISSION** (수수료제) | | rider_share = 매출 × (1 - 수수료%) |

#### 4.3.2 결제그룹 분리 (`settlement_groups`)
한국 대리업 관행: 같은 운행이라도 결제 방식에 따라 **누가 돈을 들고있는지** 다름.

| 그룹 | 색상 | 설명 |
|---|---|---|
| 기사 보유 | #d97706 (주황) | 현금/기사계좌 — 기사가 직접 받음 |
| 회사 보유 | #0f6e56 (녹색) | 회사계좌/나라시/미수/카드 — 회사가 받음 |

#### 4.3.3 정산 방향 (자동 계산)
```
balance = rider_holds (기사가 들고있는 돈) - rider_share (기사 몫)

balance > 0 → 기사가 회사에 입금 (rider_owes_company)
balance < 0 → 회사가 기사에게 지급 (company_owes_rider)
balance = 0 → 정산 완료
```

#### 4.3.4 시급제 근무시간 입력 (양양대리 패턴)
- 사장님 요구: "콜 가다가 출근, 콜 끝나고 퇴근이라 정확한 시각 모름"
- → **0.5시간 단위 직접 입력** (출퇴근 시각 입력 모드와 별도)
- 같은 기사+날짜 자동 UPDATE (덮어쓰기)
- `/attendance` 페이지에서 빠른 입력 (오늘/어제 버튼)

### 4.4 마일리지 시스템 (customer_mileage)

#### 정책 (양양대리 기준)
| 항목 | 값 |
|---|---|
| 적립률 | 운임의 **10%** (`fare_policies.mileage_earn_pct`) |
| 적립 대상 | (운임 - 마일리지 사용액) — **마일리지 사용분 제외** (마일리지 복리 방지) |
| 사용 단위 | **5,000원** |
| 회계 처리 | 옵션 A (운임은 원금으로 기록 + mileage_used 별도) |

#### 거래 종류
- **EARN**: 운행 시 자동 적립
- **USE**: 운행 결제 시 사용
- **EXPIRE**: 만료 (현재 미사용)
- **ADJUST**: 수동 조정 (사장님이 직접)

#### 양양대리 운영 현황
- 활성 고객 239명 중 **232명 (97%)** 이 마일리지 보유
- 총 누적 잔액 **1,509,000원**
- 1위: 아이리스양승창 53,500원

### 4.5 GPS 검증
운행의 신뢰성 확보를 위한 자동 검증.

- **자동 GPS 추적**: 운행 중 15초 간격으로 좌표 수집 (`auto_gps_tracks`)
- **수동 GPS 입력**: 출발/도착 지점 수동 입력 (`manual_gps_points`)
- **자동 비교**: 두 데이터를 비교해 출발지/도착지/거리/시간 일치 여부 검증 (`gps_comparisons`)
- 결과: MATCH / SIMILAR / MISMATCH 3단계
- 업체별 임계값 조정 가능 (`gps_settings`)

### 4.6 사용료 / 과금 시스템
DriveLog 자체의 SaaS 과금 모델 — **월 기본료 + 기사당 단가**.

| 요금제 | 기본료 | 기사당 | 무료 포함 | 최대 |
|---|---|---|---|---|
| 스타터 | 0원 | 10,000원 | 1명 | 5명 |
| 베이직 | 30,000원 | 5,000원 | 3명 | 20명 |
| 프로 | 50,000원 | 3,000원 | 10명 | 50명 |
| 무제한 | 100,000원 | 0원 | 0 | 무제한 |

#### 과금 보호 장치
- **기사수 초과 시 자동 차단**: 요금제 한도 초과하면 핵심 메뉴 잠금 (👥 마크 표시)
- **요금제 시즌별 특별가**: 피크/비시즌 할인 가능 (`plan_seasonal_rates`)
- **요금제 변경 이력**: `plan_change_history`, `plan_price_history`
- **노출 제어**: 테스트 요금제는 SUPER_ADMIN 화면에 비노출 (`is_visible`)

### 4.7 권한 / 통합 관리
- **역할별 메뉴 권한** (`role_permissions`): MASTER가 메뉴별로 토글
- **WEB / MOBILE / BOTH 플랫폼 분리**
- **감사 로그** (`audit_logs`): 모든 중요 작업 기록
- **문의사항** (`inquiries`): SUPER_ADMIN ↔ MASTER 채널 (RENEWAL/UPGRADE/DOWNGRADE/GENERAL/BUG)

## 5. 보안 / 신뢰성

### 5.1 인증
- **bcrypt 12 rounds** 비밀번호 해싱
- **JWT** 액세스 토큰 (15분) + **리프레시 토큰** (장기, DB 저장)
- **Helmet.js** 보안 헤더
- **Rate Limit**: 일반 API 200/15분, 로그인 20/15분, 셀프가입 10/시간

### 5.2 계정 보호
- **연속 로그인 실패 5회 → 계정 잠금** (`login_fail_count`, `locked_until`)
- **비밀번호 이력**: 최근 N개 재사용 방지 (`password_history`)
- **세션 만료**: 비활성 시 자동 로그아웃

### 5.3 데이터 보호
- **멀티테넌트 격리**: 모든 쿼리에 `WHERE company_id = ?`
- **라이선스 만료 체크**: 만료된 업체는 운행 등록 차단
- **감사 로그**: 모든 CRUD + 권한 변경 + 마일리지 조정 기록
