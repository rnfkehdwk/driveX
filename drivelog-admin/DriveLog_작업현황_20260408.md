## 🔍 주요 학습 / 함정

1. **카카오 keyword API의 형태소 매칭** — "양우내안애" 검색 시 "닥엔돈스", "양우솔어린이집" 같은 무관 결과가 섞임. 클라이언트 측 토큰 필터 (`matchesQuery`) 필수.

2. **`navigator.geolocation`은 HTTPS 필수** — `localhost` 외에는 HTTP에서 거부. 운영 환경(`https://...`)은 OK.

3. **PC 데스크톱 GPS 부정확** — `enableHighAccuracy: true`보다 `false`가 IP geo로 더 잘 잡힘. 모바일 RideNew의 GPS 출발/도착 버튼은 정확도가 필요하므로 `true`, 검색 모달은 `false` 사용.

4. **Vite 빌드 후 함수명/변수명은 minify**되지만 **문자열 리터럴은 그대로 남음**. 빌드 검증 시 `loadLastCoord` 같은 함수명 대신 `addrSearchLastCoord`(상수 값) / `"가까운 순으로 정렬"`(JSX 텍스트) 같은 문자열로 검사.

5. **React `<select>`는 프로그래매틱 펼치기 거의 불가능** — UX 검색 자동 필터링이 필요하면 native select 버리고 input + custom 드롭다운으로 가야 함. `onMouseDown`을 써야 `onBlur`보다 먼저 발화 (200ms setTimeout으로 onBlur 지연도 동시에 필요).

6. **카카오 keyword API의 `radius` + `sort=distance`** — radius 안에 결과 없으면 빈 응답. 좌표 빼고 fallback 검색이 안전.

7. **MariaDB collation 통일** — calls 테이블만 utf8mb3였어서 JOIN 시 collation 충돌. `CONVERT(... USING utf8mb4)` 또는 테이블 ALTER로 통일 필요.

8. **payment_method 영문 alias 매핑** — 양양대리는 코드체계가 001~006(한글). 프론트에서 보내는 'CASH'/'CARD'를 한글 코드로 변환하는 alias map이 백엔드에 필요.
