## 발견된 추가 이슈

### 1. SUPER_ADMIN 운행등록 500 (최소필드)
- **원인**: rides.js 수정본 미배포 (start_address만 보내면 DB 에러)
- **상태**: PC 로컬에서 수정 완료, NAS 배포 필요
- **영향**: 모바일 앱에서는 GPS 출발 시 started_at이 자동 입력되므로 실사용 영향 낮음

### 2. 업체 상태(status) 변경 API
- **재현**: PUT /api/companies/2 { status: 'ACTIVE' } → 400 "수정할 항목이 없습니다"
- **원인**: companies.js PUT에서 status 필드가 allowed 목록에 없을 가능성
- **영향**: 관리자 웹 UI에서는 별도 처리로 동작할 수 있음 — 확인 필요

### 3. 외부 DDNS 접속
- **결과**: 내부 네트워크에서 외부 DDNS 도메인 접근 실패 (Failed to fetch)
- **원인**: NAT 루프백 미설정 — 내부에서는 내부 IP로 접속해야 함
- **상태**: 정상 (외부 기기에서는 접속 가능)
